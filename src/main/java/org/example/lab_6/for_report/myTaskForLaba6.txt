Изучение словесной постановки задачи и выделение видов данных
Задача: необходимо организовать хеш-таблицу с открытой адресацией, используя специфическую хеш-функцию, и реализовать процедуру поиска по ключу в этой таблице.

Виды данных:
•	Ключи (целочисленные значения), которые будут вставляться в хеш-таблицу.
•	Хеш-таблица (массив целых чисел или null-значений для представления пустых слотов).
•	Размер хеш-таблицы M (целое число, определяющее количество слотов в таблице).
•	Коэффициент d (вещественное число, полученное из формулы (5−1)/2).

Математическая постановка задачи
1.	Хеш-функция: h(k)=trunc(M⋅frac(k⋅d)), где frac(x) обозначает дробную часть числа x, trunc(x)— отбрасывание дробной части числа x, k — ключ, M — размер хеш-таблицы, d=(5−1)/2
2.	Цель: Минимизировать количество коллизий и обеспечить эффективный поиск по ключу.
3.	Процедура поиска: Для заданного ключа k вычисляется его хеш h(k) и проверяется наличие ключа в таблице.
    Если ключ находится на вычисленной позиции, возвращается индекс этой позиции. Если слот пуст или содержит другой ключ,
    осуществляется последовательный поиск с шагом 1, пока не будет найден ключ или не будут проверены все слоты. В случае неудачи возвращается −1

Выбор метода решения задачи
Для решения задачи выбран метод хеш-таблицы с открытой адресацией:
•	Хеш-функция: Описанная выше функция используется для определения индекса вставки для каждого ключа.
•	Разрешение коллизий: Применяется метод линейного пробирования, при котором в случае коллизии последовательно проверяются следующие слоты таблицы до нахождения свободного.
•	Поиск: Используется аналогичный подход линейного пробирования для нахождения ключа, сравнивая ключи в слотах с искомым до успешного нахождения или до обхода всех слотов.
Этот метод позволяет эффективно распределять ключи по хеш-таблице и минимизировать количество коллизий, что важно для быстрого поиска и вставки элементов.
